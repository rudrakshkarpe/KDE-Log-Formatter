# Annex 5: Open standards

## 3.1.3.2: Transparency of the software product

### Requirement

* The APIs should correspond to open standards.

## Documentation

*Interface documentation, permalink to the software source code on a source code management platform, software licences or similar*

**Kate** is released as open source code under the <GPL/MIT/ETC cX> license. Its code always has been fully transparent, development is taking place in the open. All APIs are documented. Code can be inspected and modified by any user.

* [Kate API Documentation](https://api.kde.org/legacy/3.5-api/kdelibs-apidocs/kate/html/index.html)
* [Kate source code](https://invent.kde.org/utilities/kate)
* [Kate license](https://invent.kde.org/utilities/kate/-/tree/master/LICENSES)
